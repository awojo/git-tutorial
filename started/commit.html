Committing

One of the main building blocks when creating code on GitHub.

What are commits?

Commits are referred to as a record that shows the changes of the file. It is kinda like a save point in a way with a summary title and description to go along side the edit that was made from the code. Commits also provide who created the commit and the time a commit was created making up what is called the editing history (also called branch history) for the project.

How Do I Make a Commit?

To add a commit you first need to make a change to the project. This can be as simple as renaming a file to constructing an abundance of code as long as it is different from the original state the code was in. When you finish Github automatically requires you to create a commit explaining the overall process of what you did to the code. Once the commit is done you send it to the branch you used and it will be displayed on the history tab.

The left side column showcases each commit along with a description if you select it
